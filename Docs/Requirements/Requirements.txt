# Initial requirements for the QMS project

## Objective
Migrate and improve the QMS documentation system (formerly SAC) from serverjz to the Intranet, complying with ISO9001:2015 standards.

## Scope
- Review and analysis of the current system (documents and HTML).
- Design of a centralized and accessible solution on the Intranet.
- Implementation of version control and permissions.
- Compliance with ISO9001:2015.

## Functional requirements
- Secure and controlled access to documentation.
- Efficient document search.
- Change and version tracking.
- User and administrator-friendly interface.

## Non-functional requirements
- Information security and backup.
- Ease of maintenance and scalability.
- Compatibility with modern browsers.
